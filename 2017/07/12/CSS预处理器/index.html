CSS预处理器

2017-07-12 • 分类: 技术 • 标签: CSS

为什么需要css预处理器？

目标： 提供 CSS 缺失的样式层复用机制、减少冗余代码，提高样式代码的可维护性。

css预处理器有哪些？

Less，Sass，Stylus
http://less.bootcss.com/features/

基本语法

Less 的基本语法属于「CSS 风格」

.box {
    display: block;
}

嵌套语法

 .box {
    display:block;
    .box-red {
        background: red;
    }

    & .box-yellow {
        background: yellow;
    }
 }

生成的css为：

.box .box-child {
    display: inline-block;
}

备注： & represents all parent selectors (not just the nearest ancestor)

变量

@base-color: #fff;
@blue-color: @base-color - #ff0;
.parent {
  color: red;
  .child span {
    color: @blue-color;
  }
}

变量作用域

变量作用域都是按嵌套的规则集划分，并且在当前规则集下找不到对应变量时会逐级向上查找，注意这个和原生 CSS 的逻辑是完全不同的。
如果在代码中重写某个已定义变量的值，less变量的计算都会以这个变量最后一次被定义的值为准。这个行为被称为「懒加载（Lazy Loading）」。

@size: 16px;
.box-one {
    font-size: @size;
}

@size: 20px;
.box-two {
    font-size: @size;
}

转换成css为： 

.box-one {
    font-size: 20px;
} 

.box-two {
    font-size: 20px;
}

@import

针对于模块化开发
@import “foo”; // 若引入less文件，则可省略格式后缀。foo.less被引入
@import “foo.less”; // foo.less被引入
@import “foo.php”; // foo.php会被当做less文件引入
@import “foo.css”; // 被当做原生引入

混合(Mixin)

Less 的混入有两种方式：

  1. 直接在目标位置混入另一个类样式（输出已经确定，无法使用参数）；
  2. 定义一个不输出的样式片段（可以输入参数），在目标位置输出。（注：后面如无特殊说明，mixin 均用来指代此类混入。）
.info {
    font-weight: bold;
}

.highlight(@color: red) {
    font-size: 24px;
    color: @color;
}

.tips {
    .info;
    .highlight(red);
}

继承

需要为多个元素定义相同样式的时候，目的是为了复用。

.parent {
  margin: 10px 5px;
  padding: 2px;
}
.child-one {
  .parent; /* Inherit styles from '.block' */
  border: 1px solid #EEE;
}
.child-two {
  .parent; /* Inherit styles from '.block' */
  color: #333;
}

转换成css为：

.parent {
  margin: 10px 5px;
  padding: 2px;
}
.child-one {
  margin: 10px 5px;
  padding: 2px;
  border: 1px solid #EEE;
}
.child-two {
  margin: 10px 5px;
  padding: 2px;
  color: #333;
}

函数

如色彩处理、类型判断、数值计算等内置函数等。

@base: #0000ff;
@width: 0.5;

.function {
  width: percentage(@width); // returns `50%`
  background-color: spin(lighten(@base, 25%), 8);
}

转换为css为：

.function {
    width: 50%;
    background-color: #9080ff;
}

逻辑控制

.mixin(@val) when (iscolor(@val)) {
  color: @val;
}
.mixin(@val) when (isruleset(@val)) {
  @val();
}
.mixin(@val) when (default()) {
  // default() in guards acts as else
  color: #666;
}

.logic {
  .mixin(#ff0000);
}
