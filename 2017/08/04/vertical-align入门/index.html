vertical-align入门

2017-08-04 • 标签: css

1. 什么是行内框(line-box)

  • 内容区: css假设每个元素都会生成一个或多个Box，成为元素框，元素框中心有内容区，内容区外周围包括了padding, border, margin, 但是，替换元素是包括外边距，内边距，边框。
  • 行间距: 行间距是font-size和line-height的差值，被分成两半在内容区的上下。
  • 行内框: 非替换元素(比如<span>、<p>标签等)，行内框高度= line-hieght; 替换元素(比如<input>、<textarea>等)，行内元素高度=内容区高度。
  • 行高: 两行文字基线的距离
  • 行框: 一行有很多行内框，行框是包含这些行内框的最高点和最低点。
  • 基线: 不同元素的基线位置不同，整个行内框会有一个基线，行内元素的位置是基于两者基线对齐。

2. vertical-align 属性取值

说明：

  • baseline: 元素基线与父元素的基线对齐。
  • sub: 元素基线与父元素的下标基线对齐。
  • super: 元素基线与父元素的上标基线对齐。
  • text-top: 元素顶端与父元素字体的顶端对齐。
  • text-bottom: 元素底端与父元素字体的底端对齐。
  • middle: 元素中垂线与父元素的基线加上小写x一半的高度值对齐。
  • <length>: 元素基线超过父元素的基线指定高度。可以取负值。
  • <percentage>: 同<length>, 百分比相对于 line-height。
    以下两个值相对于整行：
  • top: 元素及其后代的顶端与整行的顶端对齐。
  • bottom: 元素及其后代的底端与整行的底端对齐。
    备注：

    如果元素没有基线baseline, 则以它的外边距的下边缘为基线。
    MDN实例：https://codepen.io/vickkyy/pen/EvgoOW

3. 使用vertical-align 的要求

  • vertical-align被用于垂直对齐inline元素，即display值为inline和inline-block的元素。

4. 边界和baseline


说明：

  • 行高的上下边界是红线。
  • 文字的上下边界是绿色的线，蓝色的线是baseline。
  • 左边文字的高度与行高是一致的，因此绿线和红线重合了，中间的行高是文字大小的两倍，而右边的文字行高是文字大小的二分之一。
  • 行内元素的外边缘在行高的上边缘和下边缘这个范围内对齐，如果行高小于文字的高度也无所谓。关于baseline的定义还是直接给出标准的链接

5. inline-block 元素


说明：

  • 从左到右的三副图片都是inline-block元素，不同的是，左边包含着没有脱离正常流的内容c，中间的除了没有脱离正常流的内容以外还加了overflow:hidden,右边的没有内容但是内容区还有高度。红线代表了margin-box的边界。黄色代表的是border，绿色的是padding，蓝色的是content，蓝色的线代表baseline。
  • inline-block元素的外边缘就是margin-box的边缘。
  • inline-block元素的baseline的位置要看该元素有没有处于正常流之内的内容。
  1. 在有处于正常流内容的情况下，inline-block元素的baseline就是最后一个作为内容存在的元素baseline，这个元素的baseline确定就要根据它自身来确定。
  2. 在overflow属性不为visible的情况下，baseline就是margin-box的下边界了。
    https://codepen.io/vickkyy/pen/xLGMQE
  3. 第三种情况下，baseline还是margin-box的下边界。

6. line-box

  • 有一个区域叫做line-box，垂直方向上的对齐都是发生在这个区域里面，它有baseline，有文字盒，有上下边界。
  • inline元素也有baseline和上下边界，inline元素是需要对齐的对象。
