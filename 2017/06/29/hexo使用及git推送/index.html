hexo使用及git推送

2017-06-29 • 分类: study • 标签: hexo git

创建文章及发布

step1. hexo new “文件名”

step2. vim /source/_posts/文件名

step3. hexo generate

step4. hexo server

step5. hexo deploy

创建草稿文章

$ hexo new draft "new draft"

会在source/_drafts目录下生成一个new-draft.md 文件。但这个文件不会显示在页面上，也无法通过链接访问。类似于邮件中的草稿邮件，可作为私有文件。

如果需要预览草稿文件，则需更改配置文件_config.yml：
render_drafts: true

或在启动服务时，添加参数：
hexo drafts –drafts

若需要将草稿文件变成正常文件，需执行命令：
$ hexo publish [layout] <filename>
