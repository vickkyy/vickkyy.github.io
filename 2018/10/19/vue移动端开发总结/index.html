vue移动端开发总结

2018-10-19 • 分类: vue 移动端 • 标签: vue 移动端

移动端部分

  1. 返回顶部：根据touchend事件判断document.body.scrollTop的值是否大于1000来控制返回顶部按钮的显示和隐藏。
  2. 移动端H5实现上传图片：使用H5 API中的readAsDataURL, 获取图片返回的base64编码data-uri对象，然后去请求后台的返回图片URL。在处理图片显示的时候，会有两种选择：(1) 不请求后台，直接渲染图片 (2) base64请求后台，渲染图片。此外就是删除的时候保持和数据库同步。
  3. 移动端不支持select, 需用div模拟select下拉效果。
  4. 请求参数：传递参数的时候包含#的url, 服务端有可能不识别#, 需要编码的可以使用encodeURI(), encodeURIComponent();
    二者的区别：
    encodeURI(), decodeURI(): 操作的是完整的 URI；这俩函数假定 URI 中的任何保留字符都有特殊意义，所有不会编码它们.
    encodeURIComponent(), decodeURIComponent(): 操作的是组成 URI 的个别组件；这俩函数假定任何保留字符都代表普通文本，所以必须编码它们，所以它们（保留字符）出现在一个完整 URI 的组件里面时不会被解释成保留字符了。
    更多请参考：https://www.cnblogs.com/shuiyi/p/5277233.html
    `javascript
    let url = ‘http://www.so.com?a=1&b=2&c="中文"'
    let eu = encodeURI(url)
    let euc = encodeURIComponent(url)

console.log(‘eu = ‘, eu) //http://www.so.com?a=1&b=2&c=%22%E4%B8%AD%E6%96%87%22
console.log(‘euc = ‘, euc) //http%3A%2F%2Fwww.so.com%3Fa%3D1%26b%3D2%26c%3D%22%E4%B8%AD%E6%96%87%22

5. Hybrid: 和原生交互的时候，给IOS传递参数的时候，使用window.location.href="next://", 那么IOS就可以接收到next这个参数。

  9. 移动端一般采用什么布局？移动端设计稿尺寸是怎样的？
  • 定宽布局

  1. em和rem的区别
  • em相对父级元素设置的font-size来设置大小，如果父元素没有设置font-size, 则继续向上查找，直至有设置font-size元素。
  • rem直接参照html标签字体大小，并且所有使用rem单位的都是参照html标签

  2. 移动端click 300毫秒延迟原因？

    移动端浏览器会有一些默认的行为，比如双击缩放、双击滚动。这些行为，尤其是双击缩放，主要是为桌面网站在移动端的浏览体验设计的。而在用户对页面进行操作的时候，移动端浏览器会优先判断用户是否要触发默认的行为。

  • 通过绑定窗口改变事件，监听键盘的弹出。然后去改变固定定位元素的位置。默认键盘的宽度应该是页面的2分之一。所以位移的距离改为键盘的二分之一就可以。
    window.onresize = function () {
// $('.main')就是固定定位的元素
if ($('.main').css('tap').replace('px', '') != 0) {
  $('.main').css('top', 0)
}
}
