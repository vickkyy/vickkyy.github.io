第一讲：利用DOCKER及一些内部服务提升开发效率

why docker?

背景：多项依赖（mysql, redis, nodejs, mango, nginx…）
问题：1. 不同版本同环境冲突

  1. 系统未隔离相互影响
  2. 大量的环境配置时间

Docker优点：

  1. 体积小
  2. 启动快速资源占用低
  3. 沙箱机制环境隔离
  4. 打包执行CLI自动化

Dockerfile -> 镜像（image) -> 容器（container）

Dockerfile 好处

  1. 版本管理，自动化部署
  2. 可增量更新，服用镜像块，镜像体积小

Docker compose
抛开脚本，基于yaml配置的轻量级容器编排技术

  1. 安装： pip install docker-compose
  2. 启动：docker-compose -f docker-compose yml up
