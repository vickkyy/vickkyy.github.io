CSS选择器

2020-02-18 • 分类: 技术, 前端, CSS • 标签: CSS

ID选择器

注意点:

  1. ID选择器只能对一个元素生效,同一个页面里不允许出现两个ID相同的元素
  2. 理论上ID选择器是效率最高的选择器.但是由于它只能选一个元素,特异性太高,在实际开发中也很少在CSS里使用ID选择器.
  3. 也正是因为ID选择器特异性高,所以在JS里使用ID选择器比较常见.

类选择器

注意点:

  1. 类选择器的效率也不错,和ID选择器并不会有太大的差异.所以在写CSS的时候,比较推荐用类选择器.
  2. 类选择器会选择到所有类名相同的DOM元素, 没有数量限制.
  3. 类选择器应该是样式开发中应用最多的选择器.

属性选择器

通过DOM的属性来选择该DOM节点. 属性选择器用中括号[]包裹, 比如选择所有带有href属性的标签, 就可以使用这样的选择器:

属性选择器有如下几种形式:

  1. [attr], 用来选择带有attr属性的元素, 如刚提到的a[href]

  2. [attr=xxx], 用来选择有attr属性且属性值等于xxx的元素, 如选择所有文本类型的输入框,可以用input[type=text]:

  3. [attr~=xxx], 这个选择器中间用了~=, 选择属性值中包含xxx的元素,但一定是逗号分隔的多个值中有一个能和xxx相等才行.

  4. [attr|=xxx], 这个选择器是用来选择属性值为xxx或xxx-开头的元素, 比较常用的场景是选择某一类的属性.

  5. [attr^=xxx], 匹配以xxx开头的元素,实际上就是用正则去匹配属性值,只要是以xxx开头都可以.

  6. [attr$=xxx], 这个选择器和上一个相似, 它使用正则匹配的方式来选择属性值以xxx结尾的元素.

  7. [attr*=xxx], 用正则匹配的方式来选择属性值中包含xxx字符的所有元素. 这个选择器的规则算是最宽泛的, 只要xxx是元素属性值的子字符串,这个选择器就会生效.

Tips:

  1. 属性选择器要做文本的匹配, 所哟效率也不会高.
  2. 在使用属性选择器时, 尽量要给它设置上生效的范围, 如果只用了个[href]相当于要在所有元素里找带href的元素,效率会很低.如果用a[href]会好得多,如果用.link[href]会更好.
